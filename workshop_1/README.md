## Висновки

Для фотографій краще підходе WebP формат

Для Скріншотів краще підходе MozJPEG формат

Для графічного зображення з текстом краще підходе WebP формат

Зменшення розміру зображення знижує його вагу, прискорює завантаження та зменшує використання пам’яті.

### Адаптація зображень для Retina-дисплеїв

Для адаптації зображень під Retina-дисплеї (та інші екрани з високою щільністю пікселів) слід дотримуватись таких принципів:

#### 1. Використовувати зображення в 2× або 3× роздільній здатності
- Якщо стандартне зображення 100×100 px, для Retina потрібне 200×200 px (2×) або 300×300 px (3×).
- Це дозволяє зберегти чіткість без розмиття.

#### 2. Використовувати сучасні формати (WebP, AVIF, SVG)
- **WebP** і **AVIF** мають кращу якість при меншій вазі порівняно з JPEG/PNG.
- **SVG** ідеально підходить для векторних графіків та іконок, оскільки масштабується без втрати якості.

#### 3. CSS-медіазапити для різних дисплеїв
```css
img {
    width: 100px; /* Відображається як 100px */
    height: auto;
}

@media (-webkit-min-device-pixel-ratio: 2), (min-resolution: 192dpi) {
    img {
        content: url('image@2x.png'); /* Заміна для Retina */
    }
}
```

#### 4. Застосовувати `srcset` у HTML  
```html
<img src="image.png" 
     srcset="image@2x.png 2x, image@3x.png 3x" 
     alt="Опис зображення">
```

#### 5. Оптимізація розміру файлу
* Стиснення без втрати якості (наприклад, TinyPNG або Squoosh).
* Використання `image-rendering: crisp-edges;` для піксельної графіки.

##### Додатковий CSS для покращення рендерингу піксельної графіки

```
.pixel-art {
    image-rendering: pixelated;
}
```

#### 6. Використання SVG для векторної графіки
SVG зберігає якість на будь-якій роздільній здатності.
```
<img src="icon.svg" width="100" height="100" alt="Іконка">
```
#### 7. Використання WebP для покращення продуктивності
WebP зменшує розмір файлів без значної втрати якості, що допомагає покращити швидкість завантаження сторінки.
```
<img src="image.webp" alt="Опис зображення">
```
